Base URL: /wp-json/trx-waas-wizard/v1

Authentication: WordPress Application Passwords — Authorization: Basic base64(user:app-password)

Register Site

POST /wp-json/trx-waas-wizard/v1/register-site

Registers OpenAI proxy credentials for this site. After registration, AI Helper calls are automatically routed through the WaaS platform proxy instead of using the site owner's own OpenAI token.

Calling this endpoint with the same credentials is idempotent — if credentials are already stored, the endpoint returns the current status without overwriting.

Request Body

FieldTypeRequiredDescription
app_usernamestringrequiredProxy Basic-auth username
app_passwordstringrequiredProxy Basic-auth password

Request Example

curl -X POST https://example.com/wp-json/trx-waas-wizard/v1/register-site \
  -H "Authorization: Basic dXNlcjphcHAtcGFzc3dvcmQ=" \
  -H "Content-Type: application/json" \
  -d '{
    "app_username": "waas_proxy_user",
    "app_password": "waas_proxy_secret"
  }'

Response

{
  "status":         "site_inited",
  "status_message": ""
}

Switch Skin

POST /wp-json/trx-waas-wizard/v1/switch-skin

Triggers an async pipeline: downloads the skin archive, switches the theme skin, installs required plugins via TGMPA, and imports demo data. Poll repeatedly (empty body) to track progress.

Async endpoint. The first call with skin starts the process. Subsequent calls with an empty body (or the same body) return the current status for polling. Pass "force": true to restart after rest_api_end or error.

Request Body

FieldTypeRequiredDescription
skinstringrequired (start)Skin slug to switch to (e.g. "kindergarten")
forcebooleanoptionalSet true to restart after completion or error

Start Request Example

curl -X POST https://example.com/wp-json/trx-waas-wizard/v1/switch-skin \
  -H "Authorization: Basic dXNlcjphcHAtcGFzc3dvcmQ=" \
  -H "Content-Type: application/json" \
  -d '{ "skin": "kindergarten" }'

Poll Request Example

curl -X POST https://example.com/wp-json/trx-waas-wizard/v1/switch-skin \
  -H "Authorization: Basic dXNlcjphcHAtcGFzc3dvcmQ=" \
  -H "Content-Type: application/json" \
  -d '{}'

Response

{
  "status":  "download_skin",   // current stage
  "message": ""                 // error message when status = "error"
}

Status Progression

StatusMeaning
rest_api_startRequest accepted
need_download_skinQueued for download
download_skinDownloading skin archive
switch_skinApplying skin to theme
install_pluginsInstalling required plugins
import_demo_dataImporting demo content
rest_api_endAll done ✓ Terminal

Generate Images

POST /wp-json/trx-waas-wizard/v1/page-widgets/generate-images

Schedules async image replacement for all published posts and pages using the external image-bank service (https://content-bank.themerex.net). The service URL is fixed and cannot be changed. Runs the parallel image pipeline:

  1. Build image task map (deduplication by URL)
  2. Search for matching images
  3. Submit AI generation tasks in parallel
  4. Poll & import results
  5. Apply all Elementor updates in one write per post
Async endpoint. Shares the status slot with generate-all — run only after generate-all completes or independently. Cannot run simultaneously with generate-all.

Task Runner (Internal)

POST /wp-json/trx-waas-wizard/v1/task-runner

Internal endpoint. Called exclusively by TaskQueue::dispatch() via non-blocking wp_remote_post(). Should never be called directly from external clients.

Claims the requested task atomically, restores user context, and fires the registered WordPress action hook with $task_id prepended to the stored args.

Authentication

Uses a shared secret instead of standard WP user auth. The secret is stored in WP option trx_waas_wizard_task_runner_secret and passed via X-Trx-Task-Secret HTTP header.

Request Body

FieldTypeDescription
task_idintegerID of the task to execute (from trx_waas_wizard_tasks table)

Response

{
  "success": true       // task was claimed and action fired
}

// or if task was already claimed by another process:
{
  "skipped": true
}

Task Lifecycle

  1. TaskQueue inserts task with status = 'pending'.
  2. dispatch() fires a non-blocking POST to this endpoint for each task ID.
  3. This endpoint claims the task (UPDATE … WHERE status = 'pending') — idempotent, safe for duplicate requests.
  4. do_action($hook, $task_id, ...$original_args) is fired.
  5. The action handler calls TaskQueue::complete($task_id) or TaskQueue::fail($task_id, $error).

Plugin Translation — Generate Files

POST /wp-json/trx-waas-wizard/v1/translate-plugin

Reads languages/trx-waas-wizard.pot, translates all strings into the active site locale using TRX Addons AI Helper (batches of 50), and writes the resulting .po and .mo files to the languages/ directory.

Skips automatically if the site locale is en_US or if both files already exist (use force to override).

Authentication: Requires a logged-in user with edit_posts capability.

Large .pot files may take 30–90 s due to batched AI calls. Set an appropriate HTTP timeout on the client side.

Polling Guide

Async endpoints (switch-skin, generate-all, generate-images) follow the same pattern:

  1. Start: POST with body parameters → process is scheduled via WP Cron.
  2. Poll: POST with empty body {} every 2–5 seconds → returns current status.
  3. Stop: When status is a terminal value → process is done.

JavaScript Polling Example

async function pollUntilDone(endpoint, startBody, credentials) {
  const headers = {
    'Authorization': 'Basic ' + btoa(credentials),
    'Content-Type':  'application/json',
  };

  // Start
  await fetch(endpoint, { method: 'POST', headers, body: JSON.stringify(startBody) });

  const TERMINAL = ['generate_all_end', 'generate_images_end', 'rest_api_end', 'error'];

  // Poll
  while (true) {
    await new Promise(r => setTimeout(r, 3000));

    const res  = await fetch(endpoint, { method: 'POST', headers, body: '{}' });
    const data = await res.json();

    console.log('Status:', data.status);

    if (TERMINAL.includes(data.status)) {
      return data;
    }
  }
}

PHP Polling Example

function poll_until_done(string $endpoint, string $credentials, int $max = 300): array {
    $terminal = ['generate_all_end', 'generate_images_end', 'rest_api_end', 'error'];
    $log      = [];

    for ($i = 0; $i < $max; $i++) {
        sleep(2);

        $curl = curl_init();
        curl_setopt_array($curl, [
            CURLOPT_URL            => $endpoint,
            CURLOPT_POST           => true,
            CURLOPT_POSTFIELDS     => '{}',
            CURLOPT_RETURNTRANSFER => true,
            CURLOPT_HTTPHEADER     => [
                'Authorization: Basic ' . $credentials,
                'Content-Type: application/json',
            ],
        ]);
        $response = json_decode(curl_exec($curl), true);
        curl_close($curl);

        $log[] = $response;

        if (in_array($response['status'] ?? '', $terminal, true)) {
            break;
        }
    }

    return $log;
}
